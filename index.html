AIに任せきりではなく、本番運用に耐える品質までディレクションで作り込んだポイント：

Security & Privacy

メールアドレスは平文で保存せず、HMAC-SHA256 のブラインドインデックスのみを保持。一覧検索のキーはホワイトリスト検証でSQLインジェクションを防ぎ、認証系ルートにはレート制限を追加。個人開発でも「情報を持ちすぎない」設計を徹底しました。

SEO for a SPA

SPAはクローラーに個々のアイテムを区別されにくい課題を、/items/{id} だけLaravelが受けて生HTMLに固有の title・OGP・JSON-LD を注入する構成で解決。canonical で評価を恒久ページに集約し、動的サイトマップも生成しています。

Ops & Automation

ローカルは Docker Compose（nginx / php-fpm / scheduler / MySQL / Mailpit）で本番に近い構成を再現。期限切れ・オークション解決・お知らせ削除は cron バッチで自動化し、GitHub Actions で型チェック・ビルド・両サイドのテストを回しています。

Delivery on Shared Hosting

root も常駐プロセスも使えない共用レンタルサーバーの制約に合わせ、Apache + .htaccess でAPIとSPAを振り分け、キューは同期実行に。手元ビルド→rsync の自動デプロイスクリプトと手順書まで整備し、実際に公開・運用中です。
